**Architecture:**
```
Browser (Vite React app)
  |
  POST /api/transcript { videoId, lang? }
  GET  /api/tracks?videoId=xxx
  |
Cloudflare Pages Function (same deploy)
  |
  POST https://www.youtube.com/youtubei/v1/player
  GET  {baseUrl}&fmt=json3
  |
YouTube Innertube API (undocumented, no CORS)
```

**Why a server function:** YouTube's Innertube API returns no CORS headers. Browser cannot call it directly. The Pages Function acts as a same-origin proxy.

**Innertube request shape:**
```json
POST https://www.youtube.com/youtubei/v1/player?prettyPrint=false
Headers: { "Content-Type": "application/json", "User-Agent": "<browser-like UA>" }
Body: {
  "context": {
    "client": {
      "clientName": "WEB",
      "clientVersion": "2.20240101.00.00"
    }
  },
  "videoId": "<11-char ID>"
}
```

**Innertube response (relevant parts):**
```json
{
  "videoDetails": { "title": "...", "lengthSeconds": "...", "videoId": "..." },
  "captions": {
    "playerCaptionsTracklistRenderer": {
      "captionTracks": [
        {
          "baseUrl": "https://www.youtube.com/api/timedtext?...",
          "languageCode": "en",
          "kind": "asr",
          "name": { "simpleText": "English" }
        }
      ]
    }
  }
}
```

**Transcript fetch:** GET `{baseUrl}&fmt=json3` returns:
```json
{
  "events": [
    { "tStartMs": 0, "dDurationMs": 5000, "segs": [{ "utf8": "Hello world" }] }
  ]
}
```

**After this run:** User can paste a URL, see a clean transcript, search it, toggle timestamps, copy it, and download as TXT or SRT. Deployed and live.

## Cloudflare Pages Functions (API proxy)

- [ ] **POST /api/transcript — `functions/api/transcript.ts`**
  - Accepts JSON body: `{ "videoId": "<string>", "lang": "<string|null>" }`
  - Validates `videoId` is exactly 11 chars matching `[a-zA-Z0-9_-]{11}`
  - Returns 400 with `ApiError` if validation fails
  - Calls Innertube `youtubei/v1/player` with the videoId
  - Extracts `captions.playerCaptionsTracklistRenderer.captionTracks[]`
  - If no caption tracks → returns 404 with `{ error: "no_captions", message: "..." }`
  - If video not found / private → returns 404 with `{ error: "unavailable", message: "..." }`
  - If `lang` specified, finds matching track by `languageCode`; else uses first track
  - Fetches `{track.baseUrl}&fmt=json3`
  - Parses `events[]` into flat `Segment[]`:
    - Each event's `tStartMs` / 1000 → `start` (seconds, float)
    - Each event's `dDurationMs` / 1000 → `duration` (seconds, float)
    - Concatenate all `segs[].utf8` within an event → `text` (trim whitespace)
    - Skip events with no `segs` (these are timing markers)
  - Returns `TranscriptResponse` JSON with status 200
  - Sets `Content-Type: application/json`
  - DoD: `curl -X POST http://localhost:8788/api/transcript -d '{"videoId":"dQw4w9WgXcQ"}' -H 'Content-Type: application/json'` returns valid JSON with `segments` array.

- [ ] **GET /api/tracks — `functions/api/tracks.ts`**
  - Accepts query param: `?videoId=xxx`
  - Validates videoId same as above
  - Calls Innertube, returns only caption track metadata (no transcript body)
  - Response: `TracksResponse` JSON
  - DoD: Returns track list for a known video with captions.

- [ ] **Error handling in functions**
  - YouTube HTTP 429 → return 429 with `{ error: "rate_limited", message: "YouTube is temporarily limiting requests. Try again in a moment." }`
  - Innertube fetch throws → return 502 with `{ error: "fetch_failed", message: "..." }`
  - Malformed/missing body → return 400 with `{ error: "invalid_request", message: "..." }`
  - Non-POST to transcript / non-GET to tracks → return 405
  - DoD: Each error path tested manually or via curl.

  - Button: "SRT" (in export bar)

    ```
    1
    00:00:00,000 --> 00:00:05,000
    Hello world

    2
    00:00:05,000 --> 00:00:10,000
    Next segment
    ```
  - Sequential numbering starting at 1
  - Timestamps: `HH:MM:SS,mmm` format
  - End time: `start + duration` (clamp to next segment's start if overlapping)
  - Blank line between entries, trailing newline at end
  - Filename: `{sanitized-title}_{lang}.srt`
  - DoD: File imports into VLC → subtitles display at correct times.
